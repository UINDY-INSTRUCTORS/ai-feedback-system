# EENG 340 - Generic Project Rubric

**Course**: EENG 340  
**Assignment**: Generic Project  
**Type**: project  
**Total Points**: 100

---

## Criterion 1: Introduction, background, purpose etc. (5%)

Strong introduction that provides background context and clearly states
the purpose of the project.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 2-3 | Introduction present but doesn't give good idea of background/purpose. |
| **Unsatisfactory** | 0-0 | No introduction provided. |
| **Exemplary** | 4-5 | Strong introduction with clear sense of background and purpose. |
| **Developing** | 1-1 | Very little introduction provided. |

### Satisfactory Indicators
- Basic introduction present
- Background or purpose somewhat unclear
- Limited context provided

### Unsatisfactory Indicators
- Introduction missing entirely

### Exemplary Indicators
- Comprehensive background provided
- Purpose clearly stated
- Context well established
- Motivates the project effectively

### Developing Indicators
- Minimal introduction
- Background/purpose barely mentioned

### Keywords
introduction, background, purpose, motivation, context, overview

### Common Issues
- Jumps directly into design without context
- Purpose statement too vague or generic
- No connection to course concepts or real-world applications
- Missing background on technologies used

---

## Criterion 2: Identify and Formulate problem/design goals (10%)

Clear section that identifies the problem to be solved and formulates
specific, measurable design goals and requirements (ABET 1a).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-7 | Section present that makes sense and addresses problem/goals. |
| **Unsatisfactory** | 0-1 | No section with this label. |
| **Exemplary** | 8-10 | Section with this label that seems completely reasonable with clear goals. |
| **Developing** | 2-4 | Section with this label exists but doesn't say much. |

### Satisfactory Indicators
- Problem stated
- Some design goals present
- Requirements somewhat clear

### Unsatisfactory Indicators
- Problem formulation missing
- No design goals stated

### Exemplary Indicators
- Problem clearly identified
- Design goals specific and measurable
- Requirements well defined
- Constraints identified

### Developing Indicators
- Problem vaguely stated
- Goals not specific or measurable
- Requirements unclear

### Keywords
problem, requirements, specifications, goals, objectives, constraints

### Common Issues
- Design goals too vague (e.g., 'make it work well')
- No quantitative specifications (timing, accuracy, range)
- Constraints not identified (cost, power, size)
- No prioritization of requirements

---

## Criterion 3: Develop design/solution (15%)

Viable design solution with proper justification of choices. Shows
engineering design process (ABET 1b).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-9 | Design present and makes sense but slightly flawed or incomplete. |
| **Unsatisfactory** | 0-1 | No design present. |
| **Exemplary** | 10-15 | Viable design with clear justification and trade-off analysis. |
| **Developing** | 2-4 | Design present but has serious problems or missing key elements. |

### Satisfactory Indicators
- Design mostly viable
- Some justification provided
- Minor issues or gaps

### Unsatisfactory Indicators
- Design missing
- Incomplete or non-viable

### Exemplary Indicators
- Complete, viable design presented
- Component/approach choices justified
- Alternative designs considered
- Trade-offs analyzed
- Design meets stated requirements

### Developing Indicators
- Design has serious flaws
- Poor or missing justification
- Key elements missing

### Keywords
design, solution, architecture, component selection, schematic, block diagram

### Common Issues
- No block diagram or system architecture
- Component choices not justified
- No consideration of alternatives
- Design doesn't address all requirements
- Missing schematics or diagrams

## Criterion 4: Acquire background knowledge (Data-sheets, etc.) (10%)

Evidence of acquiring and applying technical background knowledge from
datasheets, application notes, or academic sources (ABET 7a).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-7 | Mention of datasheets but relationship to design somewhat unclear. |
| **Unsatisfactory** | 0-1 | No reference to anything. |
| **Exemplary** | 8-10 | Clear reference to datasheets/papers that makes sense and is applied. |
| **Developing** | 2-4 | No reference to datasheets or papers. |

### Satisfactory Indicators
- Datasheets mentioned
- Some specifications referenced
- Limited application to design

### Unsatisfactory Indicators
- No datasheets or technical sources referenced

### Exemplary Indicators
- Datasheets referenced and cited
- Key specifications extracted and used
- Application notes or papers referenced
- Background knowledge applied to design

### Developing Indicators
- Minimal technical documentation referenced
- Background knowledge not evident

### Keywords
datasheet, specifications, application note, reference, research, documentation

### Common Issues
- Datasheets not referenced or cited
- Key specifications (voltage, current, timing) not mentioned
- No evidence of reading/understanding datasheets
- Background knowledge not connected to design choices

## Criterion 5: Design judgements (Global, Econ, Soc, Env) (10%)

Discussion of design considerations including global, economic, social,
and environmental factors (ABET 4b).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-7 | Design judgements mentioned but not fully developed. |
| **Unsatisfactory** | 0-1 | No section addressing design judgements. |
| **Exemplary** | 8-10 | Section addressing design judgements with thoughtful consideration. |
| **Developing** | 2-4 | Design judgements vague or generic. |

### Satisfactory Indicators
- Some factors mentioned
- Basic consideration given
- Could be more thorough

### Unsatisfactory Indicators
- Design judgements missing

### Exemplary Indicators
- Multiple factors addressed
- Thoughtful analysis of impacts
- Trade-offs considered
- Realistic assessment

### Developing Indicators
- Very brief or superficial
- Generic statements only
- Not specific to this design

### Keywords
economic, environmental, social, global, sustainability, cost, impact

### Common Issues
- Generic statements not specific to design
- Only cost mentioned, other factors ignored
- No consideration of environmental impact
- Social/global impacts not discussed
- Section feels like checkbox exercise

## Criterion 6: Conduct Experiments, acquire data (15%)

Strong set of experiments used to validate the design with proper data
acquisition and documentation (ABET 6a).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-9 | Set of experiments present but could be improved. |
| **Unsatisfactory** | 0-1 | No evidence of experiments. |
| **Exemplary** | 10-15 | Strong set of experiments used to validate designs with comprehensive data. |
| **Developing** | 2-4 | Missing experiments or deeply flawed approach. |

### Satisfactory Indicators
- Basic experiments conducted
- Some data acquired
- Could be more comprehensive

### Unsatisfactory Indicators
- No experiments conducted
- No validation of design

### Exemplary Indicators
- Multiple relevant experiments conducted
- Systematic testing approach
- Data properly acquired and documented
- Test conditions clearly stated
- Results validate design

### Developing Indicators
- Very limited testing
- Experiments poorly designed
- Data quality issues

### Keywords
experiment, test, measurement, data, validation, oscilloscope, multimeter

### Common Issues
- No systematic testing plan
- Missing test data or measurements
- Test conditions not documented
- No photos or scope captures of tests
- Results don't validate design claims

## Criterion 7: Conduct Simulations, make predictions (10%)

Use of simulation tools to predict behavior and validate design before
implementation (ABET 6b).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 5-7 | Some simulations present but not extensive. |
| **Unsatisfactory** | 0-1 | No simulations provided. |
| **Exemplary** | 8-10 | Numerous simulation results showing good agreement with theory. |
| **Developing** | 2-4 | Only one or two simulations, limited value. |

### Satisfactory Indicators
- Basic simulations conducted
- Limited scope
- Results somewhat useful

### Unsatisfactory Indicators
- No simulation results

### Exemplary Indicators
- Multiple relevant simulations
- Good agreement with predictions
- Simulation results inform design
- Tools used appropriately

### Developing Indicators
- Minimal simulation effort
- Results not well explained

### Keywords
simulation, SPICE, model, prediction, analysis, LTspice

### Common Issues
- No simulations performed
- Simulations not compared to experimental results
- Simulation setup not explained
- Results not used to inform design

## Criterion 8: Interpret Results, draw inferences (20%)

Strong evidence of interpreting both simulation and experimental results,
drawing meaningful inferences and conclusions (ABET 6c).

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 7-12 | Some evidence of interpreting results with basic inferences. |
| **Unsatisfactory** | 0-2 | No evidence of interpretation of results. |
| **Exemplary** | 13-20 | Strong evidence of interpreting results with insightful inferences. |
| **Developing** | 3-6 | Small amount of interpretation, mostly just stating results. |

### Satisfactory Indicators
- Basic interpretation present
- Some inferences made
- Could be more thorough

### Unsatisfactory Indicators
- No interpretation provided
- Results just presented without discussion

### Exemplary Indicators
- Thorough interpretation of all results
- Meaningful inferences drawn
- Simulation vs experiment comparison
- Discrepancies explained
- Limitations acknowledged
- Connects results to requirements

### Developing Indicators
- Minimal interpretation
- Results stated without analysis
- Few inferences

### Keywords
analysis, interpretation, discussion, inference, comparison, error, performance

### Common Issues
- Results presented but not interpreted
- No comparison between simulation and experiment
- Discrepancies not explained
- No connection to design requirements
- Limitations or errors not discussed
- Missing performance analysis

## Criterion 9: Summary/Conclusion (5%)

Strong concluding summary of findings, design outcomes, and key takeaways.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Satisfactory** | 2-3 | Summary present but lacks depth or clear conclusion. |
| **Unsatisfactory** | 0-0 | No summary or conclusion present. |
| **Exemplary** | 4-5 | Strong concluding summary of findings and design outcomes. |
| **Developing** | 1-1 | Summary extremely brief or missing key findings. |

### Satisfactory Indicators
- Basic summary present
- Could be more comprehensive

### Unsatisfactory Indicators
- Conclusion missing

### Exemplary Indicators
- Clear summary of work
- Key findings highlighted
- Design success/limitations stated
- Future improvements discussed

### Developing Indicators
- Very brief
- Key findings missing

### Keywords
conclusion, summary, findings, outcomes, future work, lessons learned

### Common Issues
- Just repeats results without synthesis
- Doesn't address whether requirements were met
- No reflection on lessons learned
- Missing or extremely brief
- No discussion of future improvements
