# PHYS/ENGR 280 - Assignment Rubric

**Course**: PHYS/ENGR 280  
**Assignment**: Assignment  
**Type**: assignment  
**Total Points**: 100

---

## Criterion 1: Abstract & Description (10%)

Abstract clearly and concisely states the project idea, results, and accuracy.
Description explains the significance of the project.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Excellent** | 7-10 | Abstract is clear and concise with accuracy indication. Description clearly conveys significance. |
| **Good** | 4-6 | Abstract present with basic merit. Description gives some idea of significance. |
| **Poor** | 0-3 | Abstract missing or lacks substantial merit. Description missing or unclear. |

### Excellent Indicators
- Project idea stated absolutely clearly
- Results stated with accuracy indication
- Significance of project well explained

### Good Indicators
- Basic idea of project is stated
- Result is stated
- Description present and gives some idea of significance

### Poor Indicators
- Project idea not clearly stated
- Result missing or stated unclearly
- Description missing or doesn't convey significance

### Keywords
abstract, description, significance, result, accuracy, purpose

### Common Issues
- Abstract too vague or overly technical
- Results stated without accuracy or convergence discussion
- Significance of computational approach not explained
- Missing connection to physical problem

---

## Criterion 2: Algorithm & Discussion (20%)

Important details of the algorithm are described clearly and correctly.
Mathematical foundations and implementation approach are explained.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Excellent** | 13-20 | All important algorithm details perfectly clear and completely correct. |
| **Good** | 7-12 | Most important algorithm details described clearly and correctly. |
| **Poor** | 0-6 | Important algorithm details missing or incorrect. |

### Excellent Indicators
- All important algorithm details present
- Presentation is perfectly clear
- Completely correct description
- Mathematical foundations explained
- Algorithm complexity discussed

### Good Indicators
- Most important details present
- Generally clear and correct presentation
- Basic understanding demonstrated

### Poor Indicators
- Important details missing or incorrect
- Presentation unclear or wrong
- Limited understanding of algorithm

### Keywords
algorithm, method, numerical, approach, iteration, convergence, complexity

### Common Issues
- Algorithm described but not explained (just lists steps)
- No discussion of why this algorithm was chosen
- Mathematical foundations not explained
- Convergence criteria not discussed
- Algorithm limitations not mentioned
- No pseudocode or mathematical formulation

---

## Criterion 3: Implementation/Code (40%)

Code is complete and functional. Easy to read and understand with embedded
documentation. Correctness validated using special cases or analytical results.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Excellent** | 26-40 | Code functions flawlessly. Easy to read and understand. Well documented. Correctness validated. |
| **Good** | 14-25 | Code functions almost perfectly. Fairly easy to read. Some documentation. Validation attempted. |
| **Poor** | 0-13 | Code missing or not functional. Nothing done to demonstrate correct operation. |

### Excellent Indicators
- Code all there and functions flawlessly
- Code is easy to read and understand
- Sufficient embedded documentation
- Code correctness validated (special case or analytical result)
- Follows best practices (PEP8, naming conventions)

### Good Indicators
- Code all there and functions almost perfectly
- Fairly easy to read and understand
- Some documentation embedded
- At least some attempt to validate correct operation

### Poor Indicators
- Code missing or not functional
- No demonstration of correct operation
- Difficult to read or understand
- Minimal or no documentation

### Keywords
code, implementation, function, NumPy, matplotlib, validation, test

### Common Issues
- No validation against known solution or analytical result
- Code works but has no comments or docstrings
- Poor variable names (x, y, arr, temp, data)
- Not using NumPy vectorization (slow nested loops)
- No test cases or example usage
- Magic numbers instead of named constants
- Functions too long or doing too many things

---

## Criterion 4: Results (20%)

Results are complete with correct units and formulas. Valid interpretation showing
understanding of both physical system and computational techniques. Clear graphical
representations with labels.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Excellent** | 13-20 | Complete results with perfect interpretation. Clear graphs with labels and formulas. |
| **Good** | 7-12 | Mostly complete results with reasonable interpretation. Graphs fairly clear with labels. |
| **Poor** | 0-6 | Results missing or incomplete. Invalid interpretation. Graphs missing or unclear. Units missing. |

### Excellent Indicators
- Results present and complete
- Units clearly given and correct
- Perfect understanding of physical system and techniques
- Clear, understandable graphs with axis labels
- Formulas provided where appropriate
- Convergence or accuracy analysis performed

### Good Indicators
- Results present and mostly complete
- Units given and correct
- Shows significant understanding
- Graphs fairly clear and mostly understandable
- Units given with clear axis labels

### Poor Indicators
- Results missing or seriously incomplete
- No interpretation or invalid interpretation
- Units incorrect or missing
- Graphs missing or indecipherable
- No formulas or incorrect formulas

### Keywords
results, data, graph, plot, units, convergence, accuracy, error, analysis

### Common Issues
- Missing axis labels or units on graphs
- Results stated without units or dimensions
- No interpretation or discussion of results
- Graphs present but not referenced in text
- No convergence test or accuracy analysis
- No comparison to analytical solution if available
- Results not formatted clearly (tables or plots)

---

## Criterion 5: Conclusion (10%)

Conclusion is present and complete, with final result clearly stated and
uncertainty/accuracy estimate included.

### Performance Levels

| Level | Points | Description |
|-------|--------|-------------|
| **Excellent** | 7-10 | Complete conclusion with clear final result and uncertainty estimate. |
| **Good** | 4-6 | Conclusion present and reasonably complete. Final result stated clearly. |
| **Poor** | 0-3 | Conclusion missing or seriously incomplete. |

### Excellent Indicators
- Conclusion present and complete
- Final result stated clearly
- Uncertainty estimate included
- Results connected to objectives
- Computational efficiency discussed

### Good Indicators
- Conclusion present and reasonably complete
- Final result stated clearly
- May lack uncertainty/accuracy discussion

### Poor Indicators
- Conclusion missing or very brief
- Final result not stated clearly
- No connection to project goals

### Keywords
conclusion, result, summary, uncertainty, accuracy, estimate, findings

### Common Issues
- Conclusion just repeats results without interpretation
- No accuracy or uncertainty discussion
- Doesn't connect back to objectives or physical problem
- Missing or extremely brief
- No reflection on algorithm performance or limitations
